# FE UI/UX 디자인 가이드

이 문서는 Memecore 프론트엔드(FE) 프로젝트의 UI/UX 디자인 및 스타일링 작업 방식을 정의합니다.

본 프로젝트는 React 기반 애플리케이션의 로직과 CSS 스타일링을 분리하여, 디자이너가 React 환경에 대한 이해 없이도 UI 스타일링에 집중할 수 있는 워크플로우를 채택합니다.

---

## 1. 프론트엔드(`fe`) 디렉토리 구조

프론트엔드 관련 모든 소스 코드는 `fe/` 디렉토리 내에 위치합니다. 주요 폴더 구조는 다음과 같습니다.

```
/fe/
├── public/              # 이미지, 폰트 등 정적 에셋
├── src/
│   ├── components/      # 버튼, 카드 등 재사용 가능한 React 컴포넌트
│   ├── pages/           # Stake, Earn 등 각 페이지를 구성하는 React 컴포넌트
│   ├── styles/          # ✅ 모든 CSS 스타일 파일이 위치하는 디렉토리
│   │   ├── main.css
│   │   └── ...
│   ├── App.tsx          # 애플리케이션의 메인 레이아웃 및 라우팅
│   └── main.tsx         # 애플리케이션의 진입점 (CSS가 여기서 import 됨)
│
└── style-guide.html     # ✅ UI 컴포넌트의 HTML 구조를 모아둔 실시간 스타일링 가이드
```

---

## 2. UI 스타일링 워크플로우

UI 스타일링 작업은 React 애플리케이션을 직접 실행하지 않고, `style-guide.html` 파일을 통해 시각적으로 확인하며 진행합니다.

1.  **스타일 가이드 확인:**
    *   `fe/style-guide.html` 파일을 웹 브라우저에서 엽니다.
    *   이 파일에는 프로젝트에 사용되는 모든 UI 컴포넌트들의 순수 HTML 구조와 클래스(class) 이름이 정의되어 있습니다.

2.  **CSS 수정:**
    *   `fe/src/styles/` 디렉토리 내의 관련 `.css` 파일을 수정하여 디자인을 변경합니다.
    *   예를 들어, 카드 디자인을 수정하려면 `_cards.css` 파일을, 내비게이션 바를 수정하려면 `_navigation.css` 파일을 편집합니다.

3.  **결과 확인:**
    *   CSS 파일을 저장한 후, 브라우저에서 `style-guide.html` 페이지를 **새로고침**하면 변경된 디자인이 즉시 반영됩니다.

---

## 3. CSS 파일 구조

모든 스타일은 `fe/src/styles/` 내에서 관리되며, 역할에 따라 파일이 분리되어 있습니다.

- **`main.css`**: 모든 CSS 파일을 `@import` 하는 중심 파일입니다.
- **`_variables.css`**: 색상, 폰트 크기 등 프로젝트 전반에서 사용되는 공통 값을 변수로 관리합니다.
- **`_base.css`**: `body`, `h1` 등 HTML 기본 요소에 대한 전역 스타일을 정의합니다.
- **`components/`**: 각 UI 컴포넌트(버튼, 카드 등)에 대한 개별 스타일 파일을 포함합니다.

---

## 4. 신규 컴포넌트 디자인 프로세스

1.  **HTML 구조 추가:** React 개발자가 새로운 컴포넌트를 개발하면, 해당 컴포넌트의 HTML 구조를 `style-guide.html`에 먼저 추가합니다.
2.  **스타일링 진행:** 디자이너는 `style-guide.html`에 추가된 새로운 구조를 확인하고, `styles` 폴더에 새로운 CSS 규칙을 추가하여 디자인을 완성합니다.
3.  **통합:** 완료된 CSS 스타일은 React 애플리케이션에 자동으로 통합되어 최종 결과물에 반영됩니다.
